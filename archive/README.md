# archive/

Completed initiatives and finished spikes. When an initiative reaches its terminal
state, move its `INIT-*.md` (and any `checkpoints/` design/handoff docs worth keeping)
here, and update the `STATUS.md` "Recently completed" row.

- `INIT-<id>.md` — the completed initiative, with its final Status Log and outcome.
  For deep initiatives, include a Process Profile (see `~/repos/ai-max/docs/operating-model.md` §14).
- `spikes/` — time-boxed discovery reports: the recommendation (abandon / shallow /
  deep) plus findings and unknowns.

Archived docs are the record for "why did we do X" — keep them readable on their own.
